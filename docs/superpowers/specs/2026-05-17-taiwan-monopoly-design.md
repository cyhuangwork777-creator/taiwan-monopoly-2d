# 台灣大富翁 — 遊戲設計規格書

## 概述

開發一款以台灣為場景的 2D 大富翁風格棋盤遊戲，參考《大富翁3》的經典玩法。玩家在台灣各地購買地產、蓋房子、收取過路費，透過策略與運氣成為最後的贏家。

**目標平台：** 網頁瀏覽器（HTML5）
**技術棧：** Phaser 3 + TypeScript + Vite
**遊戲模式：** 本機多人（1-4 位玩家，空位由 AI 控制）
**美術風格：** 可愛卡通風（Q 版角色、鮮豔配色）
**設計解析度：** 1280x720（16:9），支援視窗自適應縮放

## 1. 系統架構

### 1.1 場景結構

```
BootScene → PreloadScene → MenuScene → GameScene
                                         ↕
                              CardScene（疊加顯示卡片效果）
                              ResultScene（遊戲結束）
```

- **BootScene**：載入最小資源（loading 圖示），初始化遊戲設定
- **PreloadScene**：載入所有遊戲素材（圖片、音效、字型），顯示進度條
- **MenuScene**：主選單、玩家人數選擇、角色選擇
- **GameScene**：主要遊戲場景，包含棋盤、角色、UI
- **CardScene**：以疊加方式顯示卡片效果（機會/命運/特殊卡片）
- **ResultScene**：遊戲結束結算畫面

### 1.2 核心模組分離

```
src/
├── scenes/              # Phaser 場景
│   ├── BootScene.ts
│   ├── PreloadScene.ts
│   ├── MenuScene.ts
│   ├── GameScene.ts
│   ├── CardScene.ts
│   └── ResultScene.ts
├── core/                # 遊戲邏輯（純 TypeScript，不依賴 Phaser）
│   ├── GameState.ts         # 遊戲狀態：玩家資料、地產、回合
│   ├── TurnManager.ts       # 回合流程控制（狀態機）
│   ├── BoardConfig.ts       # 棋盤格子定義與配置
│   ├── BankSystem.ts        # 金錢交易、存款、借款
│   ├── CardSystem.ts        # 機會/命運/特殊卡片邏輯
│   ├── PropertySystem.ts    # 地產購買、升級、過路費計算
│   └── AIController.ts      # 電腦玩家決策邏輯
├── ui/                  # UI 元件
│   ├── Dice.ts              # 骰子動畫與結果
│   ├── PlayerPanel.ts       # 玩家資訊面板
│   ├── CardPopup.ts         # 卡片彈出顯示
│   ├── ActionMenu.ts        # 操作選單（購買、蓋房、使用卡片）
│   └── BankDialog.ts        # 銀行交易介面
├── renderers/           # 畫面渲染
│   ├── BoardRenderer.ts     # 棋盤繪製
│   ├── PlayerRenderer.ts    # 角色移動動畫
│   └── BuildingRenderer.ts  # 建築物渲染
├── types/               # TypeScript 型別定義
│   └── index.ts
├── config/              # 遊戲常數與設定
│   ├── gameConfig.ts        # Phaser 設定、遊戲常數
│   ├── boardData.ts         # 棋盤格子資料
│   └── cardData.ts          # 卡片資料
├── assets/              # 靜態資源
│   ├── images/              # 角色圖、棋盤、建築、卡片
│   ├── audio/               # 背景音樂、音效
│   └── fonts/               # 字型檔案
└── main.ts              # 遊戲進入點
```

### 1.3 設計原則

- `core/` 模組為純 TypeScript 類別，完全不依賴 Phaser，方便單元測試
- `scenes/` 負責串接 core 邏輯與 Phaser 渲染引擎
- 回合流程以狀態機驅動：`ROLL_DICE → MOVING → LANDED → ACTION → END_TURN`
- 遊戲狀態集中管理於 `GameState`，所有模組透過它存取與修改資料

## 3. 遊戲機制

### 3.1 金錢系統

- 起始資金：每位玩家 $20,000
- 經過或停留起點：領取 $2,000 薪水
- 破產條件：現金 + 所有地產賣出價值 < 應付金額
- 破產時地產歸還為無主空地

### 3.2 骰子系統

- 兩顆骰子，各 1-6 點，合計決定移動步數
- 擲出雙數（兩顆點數相同）：可再擲一次
- 連續三次雙數：直接送往休息站（停留一回合）
- 骰子以 sprite sheet 動畫呈現翻滾效果

### 3.3 地產系統

**購買：**
- 踩到無主地產時，可選擇購買或放棄
- 購買價格為該地的空地價格

**建築升級（共 4 階段）：**

| 階段 | 升級費用 | 過路費倍率 |
|------|---------|----------|
| 空地 | — | 1x |
| 一棟房子 | 空地價格 × 50% | 2x |
| 兩棟房子 | 空地價格 × 50% | 4x |
| 三棟房子 | 空地價格 × 50% | 8x |
| 旅館 | 空地價格 × 100% | 16x |

**同色加成：** 擁有同一等級（同色）全部 4 塊地產時，過路費額外 +50%

**賣出：** 玩家可隨時賣出地產給銀行，售價為（空地價格 + 已投入建築費用）× 50%

### 3.4 特殊格子

| 格子類型 | 效果 |
|---------|------|
| 起點 | 經過或停留領取 $2,000 |
| 機會 | 抽取一張機會卡，立即執行或獲得卡片 |
| 命運 | 抽取一張命運卡，立即執行或獲得卡片 |
| 銀行 | 開啟銀行介面：可存款（獲利息）、借款、購買卡片 |
| 休息站 | 強制停留一回合，下回合正常行動 |

### 3.5 銀行系統

- **存款**：存入銀行的錢每經過起點時獲得 10% 利息
- **借款**：可借最多 $10,000，每經過起點扣 15% 利息。欠款未還清不可再借
- **購買卡片**：花 $3,000 隨機購買一張特殊卡片

### 3.6 卡片系統

**機會卡（踩到機會格抽取，共 12 張）：**
- 獲得 $1,000 / $2,000 / $3,000
- 前進到指定地點
- 獲得一張特殊卡片
- 免費升級一次自有地產
- 所有玩家各給你 $500

**命運卡（踩到命運格抽取，共 12 張）：**
- 損失 $1,000 / $2,000 / $3,000
- 退回起點
- 繳交所有地產維修費（每棟房子 $200、每座旅館 $500）
- 送往休息站
- 所有玩家金錢重新洗牌

**特殊卡片（8 種，手動使用）：**

| 卡片 | 效果 | 使用時機 |
|------|------|---------|
| 均貧卡 | 所有玩家金錢重新平均分配 | 自己回合開始時 |
| 查稅卡 | 指定一位玩家繳交總資產 10% 稅金 | 自己回合開始時 |
| 免罰卡 | 下次被收取過路費時免除 | 被收過路費時自動觸發 |
| 購地卡 | 強制購買當前所在的他人地產（原價） | 踩到他人地產時 |
| 拆除卡 | 拆除指定地產上的一棟建築 | 自己回合開始時 |
| 翻倍卡 | 下次收取過路費時金額翻倍 | 自己回合開始時 |
| 遙控骰子 | 下次擲骰可自選 1-12 的步數 | 擲骰前 |
| 路障卡 | 在指定格子放置路障，經過的人停留一回合 | 自己回合開始時 |

**卡片持有上限：** 每位玩家最多 5 張。超過時需先使用或丟棄。

## 5. AI 對手

### 5.1 決策邏輯

AI 採用簡易策略型邏輯，根據當前局勢做出合理決策：

**購地策略：**
- 資金 > 地價 × 2 時購買（保留足夠現金）
- 如果購買後能湊齊同色全套，降低門檻至資金 > 地價 × 1.2
- 資金不足時放棄

**建築策略：**
- 優先升級同色全套的地產
- 資金 > $8,000 時考慮升級
- 從最高等級地產開始升級（投報率高）

**卡片使用：**
- 均貧卡：自己資金低於平均值時使用
- 查稅卡：對資產最高的玩家使用
- 拆除卡：對過路費最高的地產使用
- 其他卡片在合理時機使用

**銀行策略：**
- 資金充裕（> $10,000）時存款
- 資金不足且有投資機會時借款

### 5.2 AI 難度

第一版僅提供單一難度（中等策略型），未來可擴展為簡單/中等/困難。

## 8. 遊戲流程

### 8.1 開始流程

1. 進入主選單
2. 選擇玩家人數（1-4 位人類玩家）
3. 每位玩家選擇角色（已選的不可重複）
4. 剩餘角色由 AI 控制
5. 進入遊戲場景，所有角色站在起點

### 8.2 回合流程

```
回合開始
  → 顯示「XX 的回合」提示
  → 玩家可選擇使用卡片（回合開始型卡片）
  → 擲骰子（或使用遙控骰子卡）
  → 角色逐格移動動畫
  → 到達目的格
  → 觸發格子事件：
      - 空地：詢問購買
      - 自有地產：詢問升級
      - 他人地產：支付過路費（或使用免罰卡）
      - 機會/命運：抽卡並執行
      - 銀行：開啟銀行介面
      - 休息站：停留一回合
  → 檢查是否有玩家破產
  → 如擲出雙數且未連續三次，再擲一次
  → 回合結束，切換到下一位玩家
```

### 8.3 結束條件

- **淘汰制：** 僅剩一位玩家未破產時，該玩家獲勝
- **回合上限：** 達到 50 回合時，總資產（現金 + 地產價值）最高者獲勝
- 結束後顯示排名與遊戲統計

## 10. 角色圖檔處理

### 10.1 現有素材

所有原始角色圖檔位於 `腳色圖檔/` 目錄：

| 檔案 | 處理需求 |
|------|---------|
| Q版大黃蜂.png | 去除黃色背景 → 透明 PNG |
| Q版馬力歐.png | 去除棋盤背景 → 透明 PNG |
| Q版皮克敏.png | 去除淺藍背景 → 透明 PNG |
| Q版巨金怪.png | 去除米色背景 → 透明 PNG |
